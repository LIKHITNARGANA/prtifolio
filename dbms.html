Introduction to DBMS

DBMS (Database Management System) is software that allows you to create, manage, and interact with databases. It provides a systematic way to store, retrieve, and manipulate data.

A DBMS helps in managing large amounts of data efficiently, ensuring data integrity, and supporting transactions.

DBMS Concepts

Key concepts of DBMS include:

  • Data: Raw facts and figures.
  • Database: An organized collection of data.
  • DBMS: A system used to manage databases.
  • Schema: The structure of a database.
  • Entity: A real-world object represented in a database.
  • Attribute: The property of an entity.

Types of DBMS

There are four main types of DBMS:

  • Hierarchical DBMS: Data is structured in a tree-like format (e.g., IMS).
  • Network DBMS: Data is represented in a graph structure (e.g., IDMS).
  • Relational DBMS (RDBMS): Data is stored in tables (e.g., MySQL, Oracle).
  • Object-Oriented DBMS: Data is represented as objects (e.g., ObjectDB).

SQL Basics

SQL (Structured Query Language) is used to interact with relational databases. It allows you to create, read, update, and delete (CRUD) database records.

Example of a basic SQL query:

        
          SELECT * FROM Students WHERE Age > 20;

This query retrieves all records from the "Students" table where the age is greater than 20.

Normalization

Normalization is a process of organizing the data in a database to reduce redundancy and improve data integrity. The primary goal is to divide large tables into smaller ones and link them using relationships.

There are different normalization forms (1NF, 2NF, 3NF, BCNF) to ensure that a database is free of anomalies.

Transactions in DBMS

A transaction in DBMS is a sequence of operations performed as a single unit. A transaction ensures that either all operations are successfully completed or none are performed (Atomicity).

  • ACID Properties: Atomicity, Consistency, Isolation, Durability.

Keys in DBMS

Keys are used to uniquely identify records in a table. Types of keys include:

  • Primary Key: A unique key for each record.
  • Foreign Key: A key used to link two tables.
  • Candidate Key: Any key that can act as a primary key.
  • Composite Key: A key made up of more than one column.
